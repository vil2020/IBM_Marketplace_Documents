ASSET METERS

METER GROUPS

A Meter Group is the IoTSensorType

METERS

The Meter is the IoTDevice Data Point (temp, humidity etc)

A meter can be one of three types.

  1. A CONTINUOUS meter is one where the meter reading increments over time unless it is reset, or it rolls over. With a CONTINUOUS meter you need to decide whether the meter reading type is an ACTUAL or a DELTA reading when compared to the previous reading.
  2. A CHARACTERISTIC meter is used for an observation,
  3. A GAUGE meter records a numeric measurement including the history
  4. For a CONTINUOUS or GAUGE meter it is a good idea to indicate the unit of measure, this is not needed for a CHARACTERISTIC meter.

Ensure that all METERS for all Data Points are created and allocated to the correct METER GROUP and have the correct unit of Measure & Conversion.

UNITS OF MEASURE & CONVERSION

  • When creating a METER, if the unit of Measure & Conversion does not exist, then create the correct values.

Link the METER GROUP to the ASSET in ASSET Maintenance

Link the IotPOIMap->DEVICEID to the ASSET->DEVICEID in MAS

A periodic CRON job in the VI App will collate the data for each device & data point.

If the devices DEVICEID is linked to an ASSET, then the aggregated value is posted to METERS via the API.

To Configure Maximo to use Attached Documents, you must perform the following:

  • Navigate to the System Configuration > Platform Configuration > System Properties application.
  • Update the following properties
mxe.doclink.doctypes.defpath c:\doclinks
mxe.doclink.doctypes.topLevelPaths c:\doclinks
mxe.doclink.maxfilesize 20Gb
mxe.doclink.path01 c:\doclinks=https://maximo-sbx.visualiseinfo.co.uk

Check how to do this in MAS - Create a folder for the attached documents (i.e. C:\DOCLINKS)

Maximum File Size

There is a file size limit on Attached Doclinks in Maximo. The out-of-the-box value is set to 10 by default. We recommend keeping the file size below 50 MB. However, it all depends on how much bandwidth your network can handle. Therefore, we suggest testing the bandwidth and latency in a test environment.

To increase the File Size limit of Attached Documents to unlimited, please follow the steps below:

Log into Maximo - Go To System Configuration -> Platform Configuration -> System Properties.

Filter for 'mxe.doclink.maxfilesize'

Expand the 'mxe.doclink.maxfilesize' by clicking on the 'blue arrow'

In the VI App server.

sudo setfacl -R -m user:www-data:rwx /var/www/html/{instance}//uploads
